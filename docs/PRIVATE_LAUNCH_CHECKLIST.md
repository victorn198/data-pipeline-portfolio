# Private Launch Checklist

Use this checklist only when you decide to make the portfolio more visible.
Nothing here needs to be public immediately.

## Repository Readiness

- [ ] Review the current uncommitted worktree outside the portfolio package.
- [ ] Decide what stays public and what remains local-only.
- [ ] Remove weak or unfinished files before another public cleanup commit.
- [ ] Upload [assets/social-preview.png](../assets/social-preview.png) in GitHub repository settings.
- [ ] Re-check screenshots after any major UI change.

## GitHub Repository Settings

- [ ] Confirm repository description is still accurate.
- [ ] Confirm topics still match the stack and target roles.
- [ ] Keep the `website` field empty until there is a real public demo URL.
- [ ] Pin this repository only when the rest of the public profile is also clean.

## GitHub Profile

- [ ] Review pinned repositories and keep only strong ones.
- [ ] Remove abandoned or low-signal repositories from the public highlight set.
- [ ] Align bio/headline with analytics, BI, SQL, Python, dbt, and API delivery.
- [ ] Add this repository to the profile README or featured section if used.

## Resume And LinkedIn

- [ ] Add this project to the resume using only claims you can defend.
- [ ] Add the repository to LinkedIn Featured.
- [ ] Use the prepared copy in [LINKEDIN_PROJECT_COPY.md](./LINKEDIN_PROJECT_COPY.md).
- [ ] Use the short interview framing in [PROJECT_INTERVIEW_NARRATIVE.md](./PROJECT_INTERVIEW_NARRATIVE.md).

## Demo And Evidence

- [ ] Decide whether to keep the project without a live public demo.
- [ ] If you deploy it, test on a clean browser and mobile viewport.
- [ ] Capture a short demo video or GIF only after the UI is stable.
- [ ] Keep screenshots current with the actual repository state.

## Interview Readiness

- [ ] Practice the 30-second and 60-second versions of the project.
- [ ] Be ready to explain AI usage honestly and briefly.
- [ ] Be ready to explain why the desktop UX exists and what problem it solves.
- [ ] Be ready to explain why the repo is more than a dashboard.

## Do Not Do Prematurely

- Do not add a fake public URL.
- Do not pin weak repositories just to fill space.
- Do not claim production-scale ML if the project is fundamentally BI and analytics engineering.
- Do not overstate AI involvement or hide it.
